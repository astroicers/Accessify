# ASP 外部事實查證紀錄（.asp-fact-check.md）

> 本檔記錄 Accessify 專案涉及第三方 API／版本／授權／法規等外部事實的查證結果，依 ASP 鐵則「外部事實驗證防護」維護。每筆紀錄含查證日期、查證對象、結論、對應 ADR／文件意涵與再驗證條件。結論若失效（如上游套件 license 變更）須重新查證並更新本檔。

## FC-001 掃描引擎授權

- **查證日期**：2026-06-24
- **查證對象**：axe-core、pa11y、HTML_CodeSniffer（HTMLCS, squizlabs）三套掃描引擎之開源授權。
- **結論**：
  - **axe-core = MPL-2.0**（Mozilla Public License 2.0，檔案級 copyleft）。只要不修改其原始檔、且不重新授權，即可封裝於商用產品中散布。
  - **pa11y = LGPL-3.0-only**（GNU Lesser General Public License v3.0 only）。以未修改之相依形式封裝，並須能提供對應原始碼。
  - **HTML_CodeSniffer = BSD-3-Clause**（寬鬆授權，非 copyleft），保留版權與授權聲明即可。**更正先前誤標為 LGPL 之錯誤。**
  - 三引擎授權各異，文件中必須分列，不可合併標示為單一授權。
- **對 ADR-007 的意涵**：
  - 引擎係以 JavaScript 注入頁面、in-browser 執行，**非傳統動態／靜態連結**；先前「動態連結 copyleft」之框架不適用，應予移除。
  - axe-core（MPL-2.0）只要不修改其原始檔即可商用封裝；CI 應檢查 vendored axe-core 與 pinned release 為 byte-identical，以證明未修改。
  - pa11y（LGPL-3.0-only）須以未修改相依形式封裝並可提供對應原始碼。
  - HTML_CodeSniffer（BSD-3-Clause）保留版權與授權聲明即可。
- **再驗證條件**：當 axe-core、pa11y 或 HTML_CodeSniffer 任一上游套件之 license 變更（升級版本、改授權條款）時，本結論失效，須重新查證並更新 ADR-007 與本檔。

## FC-002 自動化涵蓋率 30–57%

- **查證日期**：2026-06-24
- **查證對象**：自動化無障礙掃描工具對 WCAG 成功準則（success criteria）之涵蓋率數字「約 30–57%」。
- **結論**：
  - 此「約 30–57%」之**分母** ＝「至少部分可機器測試的 WCAG 2.x A／AA success criteria」，而非全部 success criteria，亦非全部 WCAG 條款。
  - 該數字屬**業界估計，非保證**；不同工具、規則集、受測站台會有差異。
  - 「誠實涵蓋率」應以可檢查形式呈現：須存在「WCAG 準則 → 自動可判定／需人工」對應清單；報表每條 finding 標註來源引擎與涵蓋類別；涵蓋率清單與引擎規則之對齊度須達 100%。
- **再驗證條件**：以現場基準站台 fixtures 實際量測校正後，更新本數字與分母說明；當引擎規則集更新致可機器測試之準則範圍變動時亦須重新量測。
