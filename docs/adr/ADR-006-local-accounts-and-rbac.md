## 背景（Context）

地端單一組織、無網際網路。需要基本的存取控制與稽核（誰建立掃描、誰看報表），但不需多租戶，也不應引入需外網的第三方登入（OAuth 等）。原 SaaS 文件的多租戶/OAuth 不適用。

---

## 評估選項（Options Considered）

### 選項 A：本地帳號 + RBAC（admin / viewer）

- **優點**：零外部依賴、最簡穩定；密碼本地雜湊；含稽核日誌；滿足權限分離。
- **缺點**：帳號需本地管理（人數少可接受）。
- **風險**：密碼管理 → bcrypt + 鎖定策略緩解。

### 選項 B：整合內網 LDAP / AD

- **優點**：軍網常見單一登入。
- **缺點**：需現場目錄服務、額外設定與離線測試環境；複雜度高。

### 選項 C：不做認證（單一操作員）

- **缺點**：無權限分離、無個別稽核，安全/稽核不足。

---

## 決策（Decision）

採 **選項 A：本地帳號 + RBAC**（依使用者確認）。

- 角色：`admin`（管理帳號、設定、所有掃描/報表）、`viewer`（檢視掃描結果與報表）。
- 密碼：**bcrypt**（cost ≥ 12）儲存；登入失敗鎖定策略；Session 逾時。
- **稽核日誌**：登入、建立/刪除掃描任務、下載報表、變更設定皆記錄（user、action、time、ip）。
- 認證**一律採用伺服器端 session（server-side session）**，不採用 JWT／短期 token；避免引入 JWT 簽章金鑰輪替的維運面。session 狀態存於本地 DB，登出或金鑰輪替即可立即失效，不依賴外網。
- `session_timeout` 預設 **30 分鐘**，**可由 `.env` 覆寫**。
- **首位 admin bootstrap**：安裝後首次啟動需完成首位 admin 初始化，採二擇一——（a）互動式建立首位 admin 帳號，或（b）由 install 產生**一次性隨機密碼**並於首次登入**強制改密**。**嚴禁出廠固定預設密碼**（亦不得寫入 `.env.example`）。**未完成初始化前，系統處於未啟用（uninitialized）狀態**，不開放任何受保護功能。
- **LDAP/AD 留作未來選項**（非 MVP，見 ROADMAP 後段/Out of Scope）。

### Session / Cookie 簽章金鑰治理

- `SESSION_SECRET` 與 cookie 簽章金鑰於 **install 首次啟動隨機產生**（每部署唯一，嚴禁固定預設值），存為 **0600** 權限檔。
- 金鑰**納入備份範圍**，且於備份 runbook **標註為敏感資料**。
- **金鑰輪替後既有 session 一律失效**（使用者須重新登入）。
- 詳細產生／儲存／輪替與 TLS 機敏資料管理見 **ADR-008**。

## 後果（Consequences）

**正面影響：**
- 零外部依賴、權限分離、可稽核，契合穩定 + 安全 + 簡化。

**負面影響 / 技術債：**
- 帳號生命週期需本地維運；未接既有目錄服務。
