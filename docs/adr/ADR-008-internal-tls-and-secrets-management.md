## 背景（Context）

Accessify 部署於軍網內網、執行期完全無外網（air-gapped）。為保護登入憑證與 session，前端到後端須以 HTTPS/TLS 傳輸，但**不得**依賴任何對外服務（公開 CA、線上 OCSP、CRL 線上查詢、Let's Encrypt 等）。同時，伺服器端 session 與 cookie 簽章需要機敏金鑰（secrets），若沿用固定預設值將造成跨部署可預測，違反安全與穩定優先原則。本 ADR 定義內網 TLS 憑證來源、終結點與輪替，以及 secrets 的產生、儲存、備份與輪替策略，維持「零對外連線」。

---

## 評估選項（Options Considered）

### 選項 A：由 api 容器終結 TLS（預設）+ 內部 CA／自簽憑證 + 安裝時隨機產生 secrets

- **優點**：部署單元自足、最少元件、最穩定；無需額外反向代理或外網 CA；憑證與金鑰皆掛載／本地產生，完全離線；secrets 每部署唯一，杜絕可預測金鑰。
- **缺點**：TLS 終結與應用同處一容器，憑證輪替需重啟 api；自簽憑證須匯入內網信任鏈。
- **風險**：憑證到期無外網提醒 → 以本地狀態頁剩餘天數 + 閾值告警（ADR-011）與輪替 runbook 緩解。

### 選項 B：由內網反向代理（如 nginx）終結 TLS，api 走內部明文

- **優點**：TLS 設定與應用解耦、可集中管理多服務憑證；憑證輪替不影響 api 程序。
- **缺點**：新增一個須維運的元件與內網信任設定；代理↔api 段需另行保護或限制於同主機 loopback；與「最簡穩定、單部署單元」目標相違。
- **風險**：多一層元件即多一處故障與設定漂移面。

> 二擇一須在部署文件中明確記載；**預設採選項 A（api 終結）**；現場若已有標準化反向代理，得文件化改採選項 B。

---

## 決策（Decision）

採 **選項 A：由 api 容器終結 TLS（預設）**，並允許文件化改由內網反向代理終結（二擇一明確記載）。

**TLS 憑證**

- 憑證來源：掛載內部 CA 簽發之憑證；或提供**自簽產生腳本** + 匯入內網信任鏈指引。
- `.env` 新增 `TLS_CERT_PATH` / `TLS_KEY_PATH` 指向掛載之憑證與私鑰。
- **維持零對外**：停用線上 OCSP stapling 外拉與 CRL 線上檢查。
- 憑證到期：由**本地狀態頁**顯示剩餘天數 + 閾值告警（見 ADR-011）；提供輪替 runbook（步驟、重啟、驗證）。

**Secrets**

- install **首次啟動隨機產生** `SESSION_SECRET` 與 cookie 簽章金鑰；**嚴禁**在 `.env.example` 放固定預設值。
- 產生之 secrets 存於 **0600 權限**檔案，納入備份範圍，runbook 標註其為敏感資料（備份與還原須加密／受控）。
- 金鑰輪替後既有 session 一律失效（使用者須重新登入）。

---

## 後果（Consequences）

**正面影響：**
- 全程零對外連線即可完成 TLS 與 secrets；每部署 secrets 唯一，消除可預測金鑰風險；憑證到期由站內告警可控。

**負面影響 / 技術債：**
- 自簽憑證需現場匯入信任鏈；憑證／金鑰輪替需重啟 api（選項 A）；secrets 檔案與備份需以敏感資料規格保管。

**後續追蹤：**
- [ ] M7：內網 TLS 憑證佈建與輪替（ROADMAP T706）。
- [ ] M5：安裝時隨機產生 secrets、0600 儲存、納入備份。
- [ ] M5：狀態頁顯示憑證到期天數 + 閾值告警（與 ADR-011 / T507 整合）。
