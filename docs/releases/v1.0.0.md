# Accessify v1.0.0

> 首個完整交付 · First complete release — 2026-06-25

**Accessify** 是一套**地端、無網際網路（軍用網路）場域**的無障礙網頁檢測工具，對內網站台執行 WCAG 2.0/2.1（A・AA）自動檢測，產出**繁體中文（台灣）/ 英文（美國）雙語報表**（HTML / PDF / Excel）。定位為「**人工檢測前的自動化輔助**」，誠實標示涵蓋率，**不宣稱 100% 合規**。

**Accessify** is an **air-gapped (no-internet / military-network)** web accessibility scanner. It runs automated WCAG 2.0/2.1 (A·AA) checks against intranet sites and produces **bilingual (zh-TW / en-US) reports** (HTML / PDF / Excel). It is positioned as an **automation aid before manual auditing** — it states coverage honestly and **never claims 100% compliance**.

## ✨ 主要功能 · Highlights

| 領域 | 繁體中文 | English |
|------|---------|---------|
| 掃描核心 | Playwright（headless Chromium）+ axe-core + HTML_CodeSniffer 雙引擎整併去重；每請求出站白名單（SSRF 防護）；支援嚴格 CSP 站台 | Dual-engine scan (Playwright + axe-core + HTML_CodeSniffer) with dedup; per-request egress allowlist (SSRF guard); works on strict-CSP sites |
| WCAG 對應 | 規則碼→WCAG 準則、嚴重度分級、站台分數、**誠實涵蓋率**（自動約 27%，刻意非 100%） | Rule→WCAG mapping, severity, site score, **honest coverage** (~27% auto, deliberately not 100%) |
| 雙語報表 | HTML / PDF（內建 CJK 字型，離線）/ Excel；連引擎「說明」也在地化 | HTML / PDF (bundled CJK fonts, offline) / Excel; engine messages localized too |
| 本地 Portal | React Web Portal、本地帳號 + RBAC（admin/viewer）、**自身通過 WCAG 2.1 AA** | React portal, local accounts + RBAC, **the portal itself passes WCAG 2.1 AA** |
| 排程 / 追蹤 | 週期重掃排程、掃描差異（已修復/新增/未改）、站內通知 | Scheduled rescans, scan diff (fixed/new/unchanged), in-app notifications |
| 地端交付 | docker-compose 單機部署、離線安裝包、備份/還原、升級/回滾、資料保留、內網 TLS（到期監控） | Single-node compose, offline package, backup/restore, upgrade/rollback, data retention, intranet TLS (expiry monitoring) |

## 🔒 場域鐵則 · Core constraints

- **執行期零對外連線** · Zero outbound at runtime（依賴/瀏覽器/字型皆內建打包；掃描受白名單把關）
- **強制 i18n，僅 zh-TW / en-US** · Forced i18n, zh-TW / en-US only（無 hardcoded 使用者可見字串）
- **穩定優先** · Stability first（pin 版本、可重現建置、可靠離線升級 + 一鍵回滾）
- **本工具自身無障礙** · The tool's own UI is accessible（WCAG 2.1 AA）

## 🖥️ 部署形態與平台 · Deployment & platform

交付形態是 **Docker Compose 兩個容器**（`api` + `worker`，共用同一映像），**不是單一執行檔（binary）**。請分兩端理解平台支援：

This release ships as **two Docker Compose containers** (`api` + `worker`, one shared image) — **not a standalone binary**. Platform support has two sides:

| 角色 Role | 平台 Platform |
|-----------|--------------|
| **伺服器端**（跑掃描引擎的主機）· Server host | **Linux 容器**（`node:22-bookworm-slim` + 內建 Chromium）。Windows 主機需透過 **Docker Desktop / WSL2** 執行；**不提供原生 Windows 版**（穩定優先：避免多一條建置／離線打包／回滾線）。地端機房常態即 Linux server。 |
| **使用者端**（操作的人）· End user | **任何作業系統的瀏覽器**（Windows / macOS / Linux）皆可。使用者端**無需安裝任何元件**，開 `https://<主機>:8443` 即用。 |

> 一句話：伺服器跑在 Linux（Docker），使用者在 Windows 等任何系統用瀏覽器操作。
> In short: the server runs on Linux (Docker); users operate it from a browser on any OS.

## 🚀 安裝（離線）· Install (offline)

```bash
# 1) 有網建置環境：打包
scripts/package-offline.sh 1.0.0          # → dist/ 映像 tar + 部署包

# 2) 現場（斷網）主機：安裝
ACCESSIFY_TAG=1.0.0 scripts/install.sh accessify-image-1.0.0.tar.gz
scripts/verify.sh                          # 冒煙驗證
docker compose logs api | grep 'initial admin password'   # 取一次性 admin 密碼，登入後立即改密
```
登入後於「設定」頁設定**掃描白名單**（空白名單＝拒絕所有掃描，屬安全預設）。維運見 `docs/RUNBOOK.md`，驗收見 `docs/ACCEPTANCE.md`。

## ⚠️ 已知限制 · Known limitations

- **自動檢測非完整 AA**：axe/HTMLCS 僅涵蓋「自動可判定」部分（約 27%），其餘準則仍須人工複核。
  Automated checks are not full AA (~27% auto-decidable); the rest needs manual review.
- **內網 SMTP 外送通知未實作**：屬新執行期相依 + 出站路徑，待 **ADR-012** 核准後實作；站內通知已可用。
  Outbound SMTP notifications are not implemented (pending ADR-012); in-app notifications are available.
- 日誌輪替交由容器 logging driver（場域既有方案）· Log rotation delegated to the container logging driver.
